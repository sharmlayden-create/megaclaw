## 二、已实现的界面与交互

### 2.1 主屏结构

- 单 Activity，全屏 Jetpack Compose。
- 顶部留白 60dp，中部为可滚动对话区，底部为固定输入条。
- 背景：径向渐变，中心色 `#4DFFB4B4`，边缘色 `#FFF8F8`，radius 1200f。
- Insets 处理：根 Box 统一使用 `Modifier.windowInsetsPadding(WindowInsets.safeDrawing)` 一次性处理状态栏、导航栏、键盘高度，子组件不再重复添加 insets padding，避免前后台切换时底部跳动。

### 2.2 欢迎区（WelcomeHero）

- 无消息时显示：红色螃蟹矢量图标 + 欢迎文案。
- 图标带呼吸缩放动效（约 4s 周期）。
- 一旦有消息，整块通过 AnimatedVisibility 退出（shrinkVertically + fadeOut）。

### 2.3 对话列表

- LazyColumn 展示消息列表；用户消息右对齐、黑底白字气泡，AI 消息左对齐、毛玻璃白底气泡。
- 新消息到达时自动滚动到底部（LaunchedEffect + animateScrollToItem）。
- 每条消息带弹入动画（animateItem）。

### 2.4 左上角返回按钮（退出对话模式）

- 位置：主界面顶层 Box 的左上角区域，`Alignment.TopStart`。
- 布局：`start = 12.dp, top = 12.dp` 边距（insets 由根节点统一处理）。
- 显示条件：仅在 `hasMessages == true` 时可见，使用 `AnimatedVisibility`。
- 动画：进入 `fadeIn + scaleIn(initialScale = 0.9f)`；退出 `fadeOut + scaleOut(targetScale = 0.9f)`。
- 视觉：极简 `IconButton`，无毛玻璃/阴影/边框等额外装饰。
- 图标：自定义矢量图 `res/drawable/ic_custom_back.xml`（由本地 SVG 转换），24dp，tint `#1D1D1F`。
- 交互：点击后调用 `viewModel.clearMessages()`，清空消息并回到初始欢迎态/推荐卡片态。

### 2.5 底部输入框（GradientBorderInputBar）

- 整行 fillMaxWidth，左右各 24dp padding。
- 内层：彩色流光渐变边框（黄→红→紫→蓝→绿）循环动效 + 辉光透明度动效，白底圆角输入框，高度 60dp。
- 左侧：32dp 圆形螃蟹头像；中间：单行输入框，占位符「输入你的问题 ~」；右侧：有内容时发送按钮弹性出现（scaleIn/scaleOut），点击或回车发送。
- 无 0.95 缩窄，与推荐卡片区左右边界一致（24dp）。

### 2.6 推荐卡片区（SuggestionCards）

- **显示条件**：仅当当前无消息时显示；有消息后整块 AnimatedVisibility 退出（与欢迎区一致）。
- **布局**：固定 4 张卡片，单行居中，不横向滑动。卡片宽 240dp、高 220dp，间距 16dp。外层一列固定宽度（4×240 + 3×16 dp）并水平居中，列内 `horizontalAlignment = Alignment.End`。
- **「换一批」按钮**：位于 4 张卡片上方，与卡片同一 Column，右对齐，与最右侧卡片右边缘对齐。背景与卡片一致：`Color.White.copy(alpha = 0.7f)`，胶囊圆角，轻阴影 + 白边，可选 blur(0.4.dp)。内容：刷新 Icon + 11sp「换一批」文案（#86868B），点击有缩放动效（0.92f）。
- **单张卡片**：封面图来自 `painterResource(card.thumbnailResId)`，高度 135dp（16:9 比例），圆角 20dp，Crop；标题 14sp SemiBold #1D1D1F，maxLines = 2，省略号；底部沉底仅显示应用名 11sp #86868B。卡片背景 White.copy(alpha = 0.7f)，圆角 28dp，轻阴影 + 白边，点击有缩放反馈（0.95f）。
- **刷新动画**：AnimatedContent 以当前 4 张卡片为 targetState；切换时旧列表 fadeOut + scaleOut(0.95)，新列表 fadeIn + scaleIn(0.95)，220ms。
- **点击行为**：点击卡片通过 Intent.ACTION_VIEW 打开 `deepLinkUri`；若 ActivityNotFoundException 则 Toast「未安装 xxx」。

## 三、已实现的数据与逻辑

### 3.1 数据模型

- **ChatMessage**：id（UUID）、role（USER / AI）、content、timestamp。
- **RecommendCard**：title、thumbnailResId（Int）、thumbnailUrl（String）、cardTintArgb（Long）、appPackageName、appName、deepLinkUri。

### 3.2 ChatViewModel

- **messages**：可变列表，当前仅内存持有，无持久化。
- **hasMessages**：由 messages 是否为空推导。
- **currentCards**：当前展示的 4 张推荐卡片；**allCards** 为 8 条固定池（标题/应用如：15分钟全身拉伸训练-小红书、新华三MegaBookPro拆箱-哔哩哔哩、Agent Skill是什么？-豆瓣、OPENCLAW为什么这么火-哔哩哔哩等），每条带 thumbnailResId、thumbnailUrl、cardTintArgb、appPackageName、appName、deepLinkUri。
- **refreshCards()**：从 allCards 中 shuffled 取 4 条赋给 currentCards，并对每条随机替换 cardTintArgb（来自固定色板）。
- **send(text)**：将用户消息加入 messages；约 800ms 后追加一条 AI 占位消息「收到您的消息，Megaclaw 正在处理中...」（无真实 API 调用）。
- **clearMessages()**：清空 messages 列表；触发 `hasMessages = false`，从对话模式恢复到欢迎区 + 推荐卡片区。

### 3.3 本地模拟图片

- 项目根目录 **image/** 下放置 PNG（如 1.png … 5.png）。
- **syncLocalMockImages** 任务（app/build.gradle.kts）：将 image/*.png 拷贝到 build/generated/res/localMock/drawable，并重命名为 local_mock_1.png 等合法资源名；main res 的 srcDir 包含该生成目录。
- 代码中通过 **R.drawable.local_mock_1** … **local_mock_5** 引用；ViewModel 里 8 条 allCards 按条分配上述 drawable 作为 thumbnailResId。

## 四、已实现的构建与运行

- 使用 Kotlin + Jetpack Compose + Hilt；应用 id：com.megaclaw，minSdk 33，targetSdk/compileSdk 36。
- 启动入口：MainActivity + ChatScreen；Hilt 注入 ChatViewModel。
- 图标与闪退修复：AndroidManifest 使用 drawable/ic_launcher_foreground；adaptive icon 使用简化版 foreground；MainActivity 中 enableEdgeToEdge 已做 try-catch 防护（见 0313闪退bug修复.md 所述逻辑）。
- Manifest 配置：Activity 已设置 `android:windowSoftInputMode="adjustResize"`，确保系统键盘高度由 Compose 统一处理。
- 执行 `./gradlew :app:assembleDebug` 可产出 app-debug.apk；本地需配置 local.properties 的 sdk.dir 或 ANDROID_HOME。
